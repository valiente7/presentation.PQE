# Quality Starts Before the Job

An interactive, single-page reflection tool on building **workforce maturity** in biotechnology and presented at the North Carolina Community College [BioTeach Conference](https://www.ncbionetwork.org/bioteach).

## Who it's for

The BioTeach Conference brings together North Carolina's whole biotechnology community, and this tool is written to speak to all of them:

- **Biotechnology educators & faculty** — the next generation of educators preparing students for quality-driven careers
- **Industry professionals** — quality, manufacturing, validation, and regulatory practitioners
- **Workforce development & career-pathway leaders** — connecting education to employment
- **Program & department leaders** — biotech department chairs, coordinators, and administrators
- **Industry partners & employers** — those who mentor, hire, and develop new graduates

The reflection prompts use inclusive framing ("your classroom, lab, program, or workplace") so an educator, a professional, and a workforce leader can each work through it meaningfully.

## What it is

A guided, ~45-minute field journal — part of the conference's **Workforce Development & Career Pathways** strand — that walks each participant through the workshop one section at a time and turns every highlighted activity into an interactive exercise:

- Opening and journey reflections
- "What does quality culture look like?" prompt
- The **Workforce Maturity Model** — Knowledge · Skills · Behaviors · Judgment · Identity — with selectable *strongest / needs-development* choices
- Four real-world quality scenarios, each with a **"Why it matters"** reveal
- An eight-habit self-rating on the *Developing → Growing → Consistent* scale
- Group debrief, action plan, and a 30-day commitment
- Closing insights from PQE professionals and a resource list

Responses are saved automatically in the visitor's own browser, and can be printed, saved as PDF, or downloaded as a text file at the end.

## How to use it

Just open the app link above in any modern browser — desktop, tablet, or phone. No sign-in, no setup. Use **Back / Next** (or the arrow keys) to move through the sections; everything you type is kept until you clear it.

## Run it locally

Download `index.html` and double-click it. That's the entire app — no server, no build step, no installation.

## Hosting on GitHub Pages

1. Put `index.html` in this repository (the file must be named exactly `index.html`).
2. Go to **Settings → Pages**.
3. Under **Build and deployment → Source**, choose **Deploy from a branch**.
4. Select your `main` branch and the `/ (root)` folder, then **Save**.
5. Wait a minute, then open the URL GitHub shows you (also at the top of this README).

## Privacy

The app stores each person's answers with their browser's `localStorage` — on their own device only. Nothing is uploaded, shared, or collected centrally. This is intentional: it's a personal reflection journal. Collecting responses would require adding a backend, which this project does not include.

## Tech

Plain semantic HTML, modern CSS, and vanilla JavaScript in one self-contained file. No frameworks, no dependencies, no build tooling. Built with accessibility in mind: keyboard-navigable controls, visible focus states, form labels, and accessible status messages.
